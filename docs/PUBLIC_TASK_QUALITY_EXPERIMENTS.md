# QATQ Task Quality Experiments

Generated by `qatq-bench --task-quality-output`. This report runs a deterministic offline retrieval task over the selected fixture corpus: raw f32 values are grouped into 16-value records, several records are used as queries, and each codec is evaluated by whether top-1 cosine retrieval matches the original f32 corpus.

This is an end-to-end task proxy, not a language-model perplexity or downstream benchmark. It is included to verify that Phase 2 exact transport preserves task decisions and to keep lossy reference paths visibly separate from QATQ's exact product surface.

## Method

- Non-finite source values are masked to zero for this task proxy so exactness stress fixtures can be ranked with cosine similarity.
- Query records are selected deterministically from the start, quartiles, midpoint, and end of each fixture.
- `phase2-lossless` is expected to preserve top-1 decisions exactly because it reconstructs the original f32 bits for finite values.
- `turboquant-q4` and `phase1-q4` are lossy reference paths and are reported only as quality/task comparators.
