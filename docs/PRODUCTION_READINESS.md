## Current Claim

QATQ can exactly compress and restore native tensor bytes, including real
llama.cpp f16 KV-cache captures. In local Metal-backed llama.cpp runs, QATQ
verified exact restore for every exported Qwen2.5 1.5B, Qwen2.5 Coder 3B, and
Phi 3.5 mini long-context KV page and beat raw, zstd, and lz4 on every long
exported page in that evidence set. In a scoped external Rust live-migration
run, standalone QATQ preserved exact task behavior through 128 generated tokens
and transferred 14,004,990 bytes for streamed block artifacts that measured
50,331,648 raw bytes, 20,405,381 zstd bytes, and 28,739,217 lz4 bytes. That is
strong evidence for the codec and adapter path, not yet a universal superiority
claim across all models, prompts, dtypes, and cache layouts.

The production target for the initial release is storage and transfer of
exported KV/tensor state. Live GPU VRAM reduction remains experimental: QATQ now
has codec-side page metadata, bounded live-page encode/restore, scheduler
decisions, a versioned adapter trait, a verified offload store with optional
shadow-copy validation, controller helpers that enforce offload/restore/cancel
ordering and account for restore-stall budgets, event-trace verification and
`qatq-kv-bench --live-vram-live-paging-gate` for restore-before-attention
safety, operator metrics, llama.cpp export-manifest replay through
`qatq-kv-bench --live-vram-export-dir`, per-page
raw/zstd/lz4/QATQ evidence reports, prefetch-deadline simulation, and offline
tests. The current patched llama.cpp adapter can emit lifecycle and attention
telemetry, export bounded token-range pages, stage scheduler-selected page
views, and route bounded page segments through `GGML_OP_QATQ_SEGMENTED_KQV`.
The latest native live-VRAM evidence restored real llama.cpp pages exactly,
offloaded every selected cold page through QATQ with zero pass-through pages,
passed strict live-paging validation, passed MLX streaming-attention equivalence
over all configured layer/head coverage, reclaimed runtime-attested GPU K/V
residency, and beat raw/zstd/lz4 on the same page boundaries. Fresh
installed-runtime reruns cover breadth profiles, sustained Coder profiles,
bf16/f32 dtype profiles, page-size variation, and sealed metadata evidence. The
implementation and validation plan for that experimental track is maintained in
[`docs/LIVE_VRAM_REDUCTION.md`](LIVE_VRAM_REDUCTION.md).
