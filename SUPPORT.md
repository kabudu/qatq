# Support

Use GitHub issues for bugs, documentation gaps, benchmark questions, and feature
requests.

Before opening an issue, include:

- QATQ commit or release;
- operating system and Rust version;
- exact command line or Rust API used;
- tensor dtype and byte length;
- whether the payload is public, synthetic, or private runtime data;
- expected and actual result.

Do not attach private model captures, credentials, cloud logs, or proprietary
runtime dumps to public issues.
